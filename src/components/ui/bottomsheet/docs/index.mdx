
#### BottomSheetBackdrop

It is React Native's [Pressable](https://reactnative.dev/docs/pressable) component, created using [react-native-reanimated's](https://docs.swmansion.com/react-native-reanimated/) `Animated.createAnimatedComponent()` function. Animations are applied using the `entering` and `exiting` props with `FadeIn` and `FadeOut` effects.

#### BottomSheetContent

It is React Native's [View](https://reactnative.dev/docs/view) component, created using [react-native-reanimated's](https://docs.swmansion.com/react-native-reanimated/) `Animated.createAnimatedComponent()` function. The sheet slides in from the bottom using `SlideInDown` animation and slides out with `SlideOutDown`.

## Migration from v0.x (gorhom/bottom-sheet)

If you're upgrading from the previous version that used `@gorhom/bottom-sheet`, here are the key changes:

### API Changes

**Before:**
```jsx
<BottomSheet snapToIndex={1} onOpen={...} onClose={...}>
  <BottomSheetTrigger>
    <Button>Open</Button>
  </BottomSheetTrigger>
  <BottomSheetPortal
    snapPoints={['25%', '50%']}
    backdropComponent={BottomSheetBackdrop}
    handleComponent={BottomSheetDragIndicator}
  >
    <BottomSheetContent>
      <BottomSheetItem>
        <BottomSheetItemText>Item 1</BottomSheetItemText>
      </BottomSheetItem>
    </BottomSheetContent>
  </BottomSheetPortal>
</BottomSheet>
```

**After:**
```jsx
<BottomSheet
  isOpen={isOpen}
  onClose={() => setIsOpen(false)}
  onOpen={() => setIsOpen(true)}
  snapPoints={[25, 50]}
  snapToIndex={1}
>
  <BottomSheetBackdrop />
  <BottomSheetContent>
    <BottomSheetDragIndicatorWrapper>
      <BottomSheetDragIndicator />
    </BottomSheetDragIndicatorWrapper>
    <BottomSheetItem>
      <BottomSheetItemText>Item 1</BottomSheetItemText>
    </BottomSheetItem>
  </BottomSheetContent>
</BottomSheet>

{/* Separate trigger button */}
<Button onPress={() => setIsOpen(true)}>
  <ButtonText>Open</ButtonText>
</Button>
```

### Key Migration Steps

1. **Remove `BottomSheetPortal` and `BottomSheetTrigger`** - These components are no longer needed.

2. **Manage `isOpen` state externally** - Use controlled state like other gluestack components (Modal, AlertDialog).

3. **Update `snapPoints` format** - Use numbers (25, 50, 90) instead of strings ('25%', '50%', '90%').

4. **Move props to root component** - Props like `snapPoints` now go on the main `BottomSheet` component.

5. **Add components explicitly** - `BottomSheetBackdrop` and drag indicators must be added as children.

6. **Update dependencies** - Remove `@gorhom/bottom-sheet` from your dependencies. The new implementation uses `react-native-reanimated` for animations (typically already installed in most projects).

### Benefits of Migration

- **Consistency** - Follows the same patterns as other gluestack-ui overlay components
- **Better TypeScript support** - Improved type definitions and autocomplete
- **Smaller bundle size** - Removes external dependency
- **More control** - Full control over animations and behavior
- **Cross-platform** - Better web support with consistent API

## Accessibility

We have outlined the various features that ensure the BottomSheet component is accessible to all users, including those with disabilities.

### Keyboard Navigation

- `Escape` - Closes the bottom sheet when pressed
- `Tab` - Moves focus to the next focusable element within the bottom sheet

### Screen Reader Support

- The bottom sheet content is announced when opened
- Focus is trapped within the sheet when open
- Focus returns to the trigger element when closed

### Focus Management

- Initial focus can be set using `initialFocusRef`
- Final focus can be set using `finalFocusRef`
- Focus scope contains focus within the sheet
